Varnish

Internally Drupal will attempt to flush its page cache every time cron.php runs. This can mean too-frequent cache flushes if you have cron running frequently. NOTE: this cache flush is global!
Select your varnish version.
Set this to the server IP or hostname that varnish runs on (e.g. 127.0.0.1:6082). This must be configured for Drupal to talk to Varnish. Separate multiple servers with spaces.
If you have established a secret key for control terminal access, please put it here.
If Varnish is running on a different server, you may need to increase this value.
What kind of cache clearing Varnish should utilize. Drupal default will clear all page caches on node updates and cache flush events. None will allow pages to persist for their full max-age; use this if you want to write your own cache-clearing logic. Installing the Expire module will enable "Selective" clearing.
Select the type of varnish ban you wish to use. Ban lurker support requires you to add beresp.http.x-url and beresp.http.x-host entries to the response in vcl_fetch.
  • The Varnish control terminal is not responding at 127.0.0.1 on port 6082.
